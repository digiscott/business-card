## Overview

Tic-Tac-Toe++ takes a familiar game and adds a simple rule change: once enough moves have been played, the oldest mark disappears when a new one is placed. This creates a more dynamic experience where players have to think not only about where to move next, but also about which spaces are about to reopen.

The concept started as a small game idea, but it became a useful product exercise. It required clear rules, simple onboarding, thoughtful UX decisions, scalable game logic, and a path toward deployment across multiple platforms.

## The Challenge

The app needed to feel simple enough for anyone to understand quickly, while adding enough strategy to make the game feel fresh. It also needed to support future expansion across web, iOS, and Android without creating separate codebases for each platform.

Key questions included:

- How can a familiar game mechanic be changed without making the experience confusing?
- How should the rules scale across different board sizes?
- What features belong in an early prototype versus a later release?
- How can AI-assisted workflows speed up planning, documentation, and development?
- How can the technical foundation support future features like difficulty levels, multiplayer, sharing, and monetization?

---

## My Role

I approached the project as both product owner and hands-on builder. I defined the core mechanic, shaped the feature set, documented requirements, explored implementation options, and used AI tools to help generate code, clarify edge cases, and plan deployment workflows.

My work included:

- Defining the core game rules and movement-age mechanic
- Mapping how gameplay changes across 3x3, 4x4, and 5x5 boards
- Creating a prototype feature roadmap
- Planning a shared codebase for web and mobile
- Evaluating Expo and React Native as the technical foundation
- Drafting prompts and implementation guidance for coding agents
- Thinking through future features like AI difficulty, sharing, analytics, and monetization
- Considering deployment needs for both the Apple App Store and Google Play Store

---

## Product Thinking

The core product decision was to keep the game familiar while introducing one meaningful twist. The mechanic needed to be understandable in seconds, but strategic enough to reward replay.

The early product direction focused on:

- A fast first-play experience
- Clear visual feedback for aging moves
- Simple board-size options
- Local play as the initial experience
- Expandable support for AI opponents and multiplayer
- Lightweight sharing after a completed game

This helped keep the prototype focused while leaving room for future growth.

## UX Considerations

The biggest UX challenge was making the disappearing-move mechanic obvious without over-explaining it. Players need to understand which move is oldest, which mark will disappear next, and how that affects their strategy.

UX considerations included:

- Showing the age of each move visually
- Highlighting the next mark that will disappear
- Keeping the interface friendly and minimal
- Explaining the rule change with short onboarding text
- Making board-size changes easy to understand
- Supporting both quick casual play and more strategic replay

---

## AI-Assisted Workflow

This project became a practical example of using AI as a product and development accelerator. AI support helped move quickly through planning, documentation, technical exploration, and implementation prompts.

AI was useful for:

- Turning a rough idea into structured requirements
- Exploring edge cases in the game rules
- Drafting implementation prompts for coding tools
- Creating deployment checklists
- Comparing technical approaches
- Planning GitHub Actions for repeatable build and deployment workflows
- Generating concise product documentation for future contributors or agents

The value was not just speed. The value was creating clearer structure around the work so decisions could be made more confidently.

The project produced a clear product direction and development plan for a cross-platform game prototype. It also became a useful example of how I work: identifying a product opportunity, clarifying the user experience, breaking down the work, and using AI as a practical accelerator rather than a replacement for product judgment.

Key outcomes included:

- A defined game concept with scalable rules
- A prototype direction for web and mobile
- A technical path using a shared codebase
- A roadmap for future enhancements
